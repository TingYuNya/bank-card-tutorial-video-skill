# Privacy and Compliance Guide

## 适用范围

这份说明面向银行卡、银行账户、支付工具和金融 App 教程。它用于降低素材处理和公开发布过程中的隐私风险，不替代法律或合规意见。

## 禁止提交到仓库的内容

1. `.env`、API Key、Access Token 和私钥。
2. 完整银行卡号、CVV、PIN 和有效期组合。
3. 短信验证码、邮件验证码和登录密码。
4. 银行账户号、routing number、IBAN 和客户编号。
5. Session Token、二维码、条形码和身份文件。
6. 未脱敏账单、交易记录、地址、手机号和邮箱。

## 图片审核

每张进入成片的图片应记录：

1. 图片文件和场景编号。
2. 已发现的敏感信息类型。
3. 遮挡区域的归一化坐标。
4. 遮挡方式，例如模糊、实色覆盖或裁切。
5. 审核状态和人工确认记录。

建议默认仅展示卡号后四位。需要展示 BIN、账户类型或地区信息时，应只保留完成讲解所需的最小范围。

## 第三方 TTS

将配音稿发送给 TTS 服务前，应确认稿件中没有：

1. 真实姓名与账户信息的组合。
2. 卡号、账户号、验证码或密码。
3. 未公开的银行内部资料。
4. 无权发送给第三方的个人信息。

## 事实核验记录

金融陈述应保存：

1. 原始陈述。
2. 修订后的表述。
3. 官方来源。
4. 核验日期。
5. 适用地区、账户条件和渠道。
6. 风险等级与发布建议。

## 公开发布前检查

- [ ] 已确认全部截图完成脱敏。
- [ ] 已检查视频每个场景的关键帧。
- [ ] 已核验当前费率、资格、押金和还款规则。
- [ ] 已标明必要的地区、渠道和时间条件。
- [ ] 已确认图片、音乐、字体和声音的授权。
- [ ] 已删除日志、缓存和中间文件中的敏感信息。
- [ ] 已在视频说明中注明信息核验日期。
