# Security Policy

## 支持版本

当前维护版本：

| 版本 | 安全更新 |
| --- | --- |
| 0.1.x | 支持 |
| 更早版本 | 不保证 |

## 敏感信息

请勿在 Issue、Pull Request、Discussion、日志或示例素材中提交：

1. API Key、访问令牌、私钥和 `.env` 文件。
2. 完整银行卡号、CVV、有效期组合和 PIN。
3. 短信验证码、登录密码和 Session Token。
4. 银行账户号、routing number、IBAN 和客户编号。
5. 身份文件、详细住址、账单和未脱敏交易记录。

发现敏感信息泄露后，应立即撤销对应凭据，并清理 Git 历史、缓存、Release 附件和构建产物中的副本。

## 私密报告

请使用 GitHub 仓库的 Private vulnerability reporting 功能提交漏洞。报告中建议包含：

1. 受影响版本和组件。
2. 复现步骤与最小示例。
3. 预期行为和实际行为。
4. 风险范围和可能影响。
5. 已验证的缓解措施。

请勿在公开 Issue 中披露可以直接利用的漏洞、访问凭据或真实金融数据。

## 第三方服务

项目可以连接 ElevenLabs、OpenAI 和 Azure Speech。使用者需要自行审查第三方服务的数据处理政策、费用、配额和适用地区，并避免向服务发送不必要的个人金融信息。

## 安全边界

隐私审核、事实核验和自动遮挡用于降低风险，无法替代人工审查。公开视频发布前应检查每个场景的关键帧和最终成片。
