# Contributor Covenant Code of Conduct

## 我们的承诺

项目维护者和贡献者承诺，为所有参与者提供尊重、专业且不受骚扰的协作环境。参与者的经验水平、身份背景、语言、所在地和技术栈不应成为被排斥或贬低的理由。

## 期望行为

参与项目时，请遵守以下原则：

1. 针对代码、文档和设计进行具体讨论。
2. 清楚说明复现步骤、适用版本和测试环境。
3. 尊重不同实现路径，对技术分歧给出可验证依据。
4. 发现安全或隐私问题时使用私密报告方式。
5. 避免提交真实银行卡、账户、身份或访问凭据。

## 不可接受的行为

以下行为可能导致内容被删除、讨论被锁定或参与权限被限制：

1. 人身攻击、侮辱、歧视、威胁或持续骚扰。
2. 未经允许公开他人的个人信息或金融信息。
3. 在 Issue、PR、日志或示例中发布 API Key、Token、完整卡号、CVV、验证码或账户凭据。
4. 故意提供误导性金融信息，或将未经核验的内容表述为官方结论。
5. 重复发布广告、推广链接或与项目无关的内容。

## 执行

维护者可以对违反本准则的内容进行编辑、隐藏、删除或锁定，并根据严重程度限制参与权限。安全漏洞、隐私泄露和凭据暴露应优先处理。

## 适用范围

本准则适用于仓库中的 Issue、Pull Request、Discussion、代码评审和项目相关的公开交流渠道。

## 归属

本文件参考 Contributor Covenant 2.1 的结构，并结合本项目的金融隐私风险进行了调整。
